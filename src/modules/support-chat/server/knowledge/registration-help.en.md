# Registration Help

## registration-create-account

Users create or access an Infinisimo account through the sign-up or sign-in pages.

After sign-in, the app connects the authentication account to the Infinisimo profile used by marketplace features.

## registration-complete-profile

Users should complete the general profile before booking or becoming a provider.

The general profile includes first name, last name, username, email address, address, country, and language.

## registration-why-location-is-needed

Location is used so customers can find providers in local searches and so the app can support location-aware marketplace behavior.

If a user wants to become a provider, the general profile location is required before provider setup can continue.

## registration-address-selection

When entering an address, users should choose a complete address from the address suggestions.

The app may require a full street address with a house number. If the address is typed manually without selecting a suggestion, the profile form may ask the user to select an address from the suggestions.

## registration-username-rules

Usernames must follow the app's validation rules. They must be at least 3 characters and less than 63 characters.

They may contain letters, numbers, dots, underscores, and hyphens. They must start and end with a letter or number, and they cannot contain consecutive separators.

## registration-profile-save-failed

If profile saving fails, the user should check required fields, address selection, and validation messages, then try again.

Support chat can explain the general requirements, but it cannot inspect why a specific profile save failed.

## registration-language-preference

Users can select a language in their profile. The route language remains the active language for the current app route, while profile language is a saved preference.
