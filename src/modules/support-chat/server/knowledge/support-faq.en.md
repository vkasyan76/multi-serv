# Support FAQ

## faq-what-can-support-chat-answer

The support chat can explain general Infinisimo usage, registration, provider setup, booking, payment, cancellation, dispute, and Terms of Use topics.

It can give plain-language guidance based on approved support content, but it cannot inspect a user's live account, order, payment, invoice, provider profile, or Stripe status.

## faq-what-if-i-need-account-help

If a question depends on a specific account, order, payment, invoice, cancellation, or provider profile, the user should use the relevant app page or contact support.

Support chat should explain the general rule and then direct the user to the right next step. It must not say that it checked live account data.

## faq-customer-start

Customers usually start by creating an account, completing their profile, browsing providers, selecting available booking slots, choosing a service for each selected slot, adding a payment method when required, accepting Terms of Use, and submitting a booking request.

The current booking flow creates a booking request first. The provider must confirm the request before it becomes a scheduled booking. Payment is handled later in the service lifecycle when payment is requested.

## faq-provider-start

Providers usually start by creating an account, completing the general profile and location, accepting the Terms of Use, creating a provider profile, setting service categories and pricing, completing Stripe onboarding for payouts, and adding availability in the calendar.

Customers can only book providers who have visible available slots and are ready for booking.

## faq-provider-not-ready

If a provider is not ready to take bookings, customers may see a message that the provider is not ready yet.

Common reasons include incomplete provider setup, missing payment/payout readiness, or no available future calendar slots. Support chat can explain these general possibilities, but it cannot inspect the provider's live account setup.

## faq-something-failed

If something fails, the user should first retry the action, refresh the page if appropriate, and check whether required profile, payment, or Terms steps are complete.

If the error is about a specific order, payment, cancellation, Stripe account, invoice, or booking record, support chat should not guess the cause. It should provide the general rule and direct the user to support or the relevant app page.

## faq-platform-role

Infinisimo is an intermediary marketplace. It provides the platform for customers to discover providers, request bookings, and, where enabled, pay providers.

Services are performed by independent providers, not by Infinisimo. Providers are responsible for the quality, legality, safety, and performance of their services.
